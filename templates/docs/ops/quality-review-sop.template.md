# Quality Review SOP

Use this SOP to review page quality for `{{PROJECT_NAME}}` after refreshes and before major launches.

## Review Goals

- Catch thin or repetitive page bodies
- Catch navigation dead ends and orphan-prone templates
- Catch unsupported EEAT or methodology claims
- Catch ranking output that reads like noise instead of guidance

## Sampling Frame

- Review at least one top-ranked entity page
- Review at least one median entity page
- Review at least one edge-case or low-confidence entity page
- Review the default category hub
- Review both representative and full ranking pages

## Checks

- Headline and summary read naturally and match the data shown
- Metrics and supporting copy do not contradict each other
- FAQ answers add decision value instead of paraphrasing the page intro
- Next-step cards send users into relevant comparisons or hubs
- Canonical, title, and description fields still match page intent

## Escalation Conditions

- Repeated copy patterns spread across many pages
- Ranking churn changes the narrative of the site without a source change
- A route family starts producing pages that no longer justify indexing

## Output

- Add findings to `docs/data-audit-log.md`
- Open a remediation backlog grouped by `copy`, `data`, `navigation`, or `indexing`
