## Project Rules

- This is a tarot reference site with a static-exported route surface and image-heavy card delivery.
- Work in this order: data -> build/runtime -> UX -> SEO -> trust/compliance -> performance -> conversion -> docs.
- Keep route helpers and tarot card data aligned so links, sitemap output, and exported pages stay in sync.
- Keep indexing layered and controlled.
- Treat `data/tarot-cards.json` and `src/lib/site.ts` as the runtime source of truth for public identity.
- Keep the skill surface small. Prefer workflow docs and `docs/solutions/`.
